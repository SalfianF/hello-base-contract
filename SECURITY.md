# Security Policy

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in any contract:

1. **Do NOT open a public issue**
2. Email: security@example.com
3. Include:
   - Contract name and affected function
   - Description of the vulnerability
   - Steps to reproduce (test contract preferred)
   - Suggested fix (optional)

## Response Timeline

| Severity | Initial Response | Fix Deployed |
|----------|-----------------|--------------|
| Critical | 24 hours | 7 days |
| High | 48 hours | 14 days |
| Medium | 72 hours | 30 days |
| Low | 7 days | 60 days |

## Scope

The following are in scope:

- All Solidity contracts in `contracts/`
- Deployment scripts in `scripts/`
- Hardhat configuration

## Out of Scope

- Issues in dependencies (report to upstream)
- Phishing attacks targeting users
- Private key mismanagement

## Bug Bounty

This is a learning project — no formal bounty program yet. But we'll credit you in the README and commit history. 🙏
